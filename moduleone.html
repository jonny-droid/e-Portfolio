Unit 2: Introduction to Computer Architecture and the Functionality

This unit aims to show that binary data (bit) is processed by physical layers of computer hardware. At the lowest level all information is encoded as 0s and 1s. These are called bits (short for binary digits). 0/1s are just one way of representing them, could be characters in the alphabet or punctuation marks. True/false is another good way to imagine bits. Memory organisation In main memory essentially all bits are one long row and that is why an address is needed. RAM is called Random Access memory because it has the ability to access cells in any order. Size of memory measured in 1024 bits (called kb as near 1000) Note: main memory known as RAM.
A sequence of bits may represent instructions or data.

Main parts of the CPU are: control unit (CU) (refers to location of next instruction), arithmetic logic unit (ALU) (doing the operation on the dat), registers (temporary storage of data), cache, buses and clock

High level programming language statements are mapped down to low level machine instructions, e.g. a Python statement could map to hundreds of low level instructions

Virtualisation provides a way of getting aroudn the constraints of software interaction with different operating systems, software etc...

Unit 5: Software as a Tool - Its Functionality and Development Methodologies

One of the problems iwth the software rield traditionally is that software systems are built from th group up and don't rely on already built components from elsewhere (as opposed to modern engineering)

One way of clasifying softare is to divide it into system software (for example the OSs and firewalls) and application software.

There are various software development methodologies. For example:
The waterfall model: software development flows in one direction. An indepth specification is written at the start. Plan > Build > Test > Review > Deploy > Maintain
This method is now largely seen as outdated.

Agile methods are now commonly used. These are adaptive methods, and is not a one way process(for example, the specification may be constantly adapted whilst writing the software. Prototypes may be made at various steps). The plan at the start minimal
Examples of agile methods include Extreme Programming (XP) and Scrum. Scrum involves several sprints (bite size pieces of a few weeks). After a sprint review things, and may even rewrite specification. Three main roles: product owner (defines featurs), scru master (leads the team) and the team itself (developrs and tests)

Types of software documentation:
Technical documentation: how software should be installed, similar to how mechanic has a car manual.
System documentatin: Describes internal mechanics of teh system needed for future maintenance. Means that other people with the software development team can understand the software.
User documentation: To show the end user how to use the software
(Sometimes the user documentation and technical documentation is blurred, as the person installing the software might also be the end-user)
