NoSQL - example of Key-Value storage Using key-value NoSQL databases involves records being stored in pairs (one key to one value). A key is an arbitrary string, for example a file name. A value can be anything as short as a single number or as long as multiple lines of code (such as those stored in a .html file). The value can also be a list. According to Davoudian et al (2018) there are a range of data storage needs where a key-value relationship is the best solution. Examples of these include shopping carts and session management. where a single key is needed to access data. In these situations key-value databases are the simplest and most scalable method of storage. Key-value databases do not use queries and instead rely the following commands: “get” (i.e. retrieve key-value), “put” (i.e. add a key value) and “delete”. Filip and Cegan (2020) write that key-value databases often do not support searches based on values, and even where this is possible, if searching by value is likely required then another type of data storage is preferable. Originally key-value databases were made with the premise that each key contains one piece of data. With Redis one key can contain millions of other keys, Davoudian et al (2018) argues that this blurs the line of what a key-value database is. Weise (2015) writes that key-value stores work well as part of a wider database management system, including both SQL and noSQL storage.
