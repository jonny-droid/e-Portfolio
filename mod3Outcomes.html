Learning Outcomes

Identify and manage security risks as part of a software development project.

During this module I was introduced to OWASPS Top 10 Web Application Security Risks. Although I was previously aware of these, by thinking about how our group application mitigated each of these threats in turn, made me understand the proactive steps needed. One example of this was mitigating against A03:2021-Injection. We were cautious any time there was an input into our system from a user. Therefore we put the following into place:
  • For image uploads we restricted these to only the following file types: .jpg, .jpeg, .png, and .gif. This avoided any injection through executable files
  • We limited the size of the file to mitigate against DDOS attacks (for example uploading files with extremely large amounts of data)
  • We needed to be confident that input boxes escaped characters, so we used SQLAlchemy due to its built in methods for this (among other reasons, such as working well with our MVC pattern)
I did further details into particular security areas:
  • Writing a discussion post on cross-site scripting.
  • Writing a a blog post on managing security risks
  • The particular risks, and mitigations, associated with each programming language (looking particulary at C and Python).
    This took the form of a team discussion and a programming activity on buffer overflows.
  • ReDoS (a form of DDOS attack exploiting Regex vulnerabilities).
  • Cryptography
Overall my thoughts were how there is a continous arms race between those responsible for applications and those wanting to exploit them. With each new technology released there are almost instant weaknesses that can be exploited (usually through poor coding or networking practices).
