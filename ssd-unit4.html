Unit 4 artifact

1) What is ReDOS and what part do ‘Evil Regex’ play?

Regular Expression Denial of Service is a type of DDOS attack. This exploits a vulnerability in Regex where, if written poorly, the system can be overwhelmed with an input. Essentially, someone enters an input that takes Regex an extended time to calculate due to a high level of complexity (Weidman, N.D.).

2) What are the common problems associated with the use of regex? How can these be mitigated?

As mentioned above, ReDOS is potentially a security issue with regex. This can be mitigated by using an Automatic Checking of Regular Expressions (ACRE) (Larson, 2018) and using EGRET (a tool which generate regular expressions) (Larson & Kirk, 2016). Larson and Kirk (2016; 2018) however state that these tools cannot be completely relied upon, as certain bugs were missed when they tested them, therefore cannot be a substitute for an experienced developer.

Bai et al (2021) describe a way of detecting regular expressions that take an extended time to calculate and sandboxing these.

3) How and why could regex be used as part of a security solution?

Regex is part of a security solution as it can be used to enforce security standards. For example, it can be used to make sure that passwords of a particular length and have at least one symbol, one uppercase and one lowercase letter. Li (2020) talks about how Regex is used within malware scanning.
