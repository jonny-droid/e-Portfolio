Glossary

Here are a list of some of the key terms from the course.

A

Abstraction (programming): reducing a complex problem to a simplified version. In OOP, it relates to only focussing on the necessary deatils, for example, abstract class can be created to supply what is needed to subclasses (even when the class is not used in itself. Can also refer to abstracting away the details for only the entities that need to see it.

Artificial Intelligence (AI): When computers perform a task which is commonly associate with a human being.

Assembly language:

a low-level programming language that is used to communicate directly with a computer's hardware.

B

Bridge:Connects 2 networks to become as one. People won't feel like moving between LAN 1 and LAN 2, for example. Only works where on the samet ype of network.

Byte: Memory is usually organised into cells (bytes). A cell = 8 bits. A byte is represented by two hexidemicals (as one hexidecimal represents 4 bits). A cell usually has a high to low end order. High end (more significant bits) > 01001101

C

Cache: a tiny bit of RAM exists in the CPU (called SRAM - much faster than normal RAM).

Casting:converts data from one type to another, e.g.an integer to a string.

Code refractoring: A Dev Ops concept for editing and cleaning up previously written code without changing how it functions.

Computational artefact: anything created by a human using a computer (e.g. a programme, video, web page, image etc…).

Computer Aided Software Engineering (CASE) tools: software tools to simplify the software development process (like how CAD is used in engineering). IDE's are an example of a CASE tool

Cyclometric complexity:worked out through counting the number of decision points. Usually better to have a lower cyclometric complexity. A hihg score may indicate that refractoring is necessary.

D

Data mining: differs from database enquiry in that it looks to identify unique patterns, rather than just retrieving stored facts.

Distributed computing:Multiple computers on the same network work together through passing messages to achieve a common goal.

Double strength encryption:using multiple keys so that if someone intercepts one key then they can’t decrypt the data.

E

Entity: a thing of interest to an organsiation which has data recorded. Forhttps://getbootstrap.com/ example a student. An entity usually takes the form of a table

Encapsulation: data hiding/an object should only revewal its essential details. Also can refer to the collection of related data and procedures into a class. Encapsulation is the process and abstraction is what is being facilitated

F

Flip flop: a fundamental unit of computer memory. Remains a zero or one until told otherwise. A modern computer chip contains millions of flipflops

G

Garbage collection:automates teh release of memory that it used and no longer needed back to the system. Python, and some other high level programming languages do this.

Gate/Logical gate: a device which produces a booloan operation.

Gateway:Generic term. Could be a router, a special type of bridge or other device. Connects two different networks that would otherwise be incompatible.

H

Hexadecimal notation: This takes advantage of the fact that bit pattens usually have length of multiples of 4. A hexadecimal notation uses a single symbol to represent a pattern of four bits. Hexadecimals are usually prefixed by “0X”.

HTML:Essentially a way of intructing browser how to display a webpage toa user. "Hypertext" means taht it it it contains tags describing how the document is displayed and links ot other documents.

Hybrid crytopgraphyusing multiple approaches/multiple cyphers

Inheritence:DRY (Don't repeat yourself). See OOP module

Internet: A worldwide network of networks.

L

Latency:the time a system takes from a request to a response. E.g. the response time for an ennd user from a web application. The lower the latency the faster the time.

Levels of abstraction: hierarcy with machine code being at the lowest level (think of lowest level being closest to electronics).

M

MAC address: A unique address for the network adapter on your device.

Machine language: collection of machine instructions consisting of binary or hexidecimal which the computer can respond directly to.

Motherboard:the primary device connecting the innards of System Unit together (.e.g. CPU, RAM etc…)

Mutibility:Whether objects once created can be changedSee OOP module

N

Networking: How computers are linked together to share information.

O

P

Packets: Blocks of data. Treated as unrelated messages, and may travel a completey different route, until they reach the transport layer at their final destination.

Parallel computing:Uses mulitple processes. The workload is divded between more than one processor. A single computer with processors connected through a bux (used heavily in supercomputers))

Polymorphism: A method can be activiated and modified by a subclass/child class.

Pointers: used by assembly langauges. Is an address for the variable's location in memory Pointers: Poison package:when installed it achieves a functionality but also performs a negative behaviour(e.g. information leakage). An example might be from a library you install. Programming language: is used for people to express algorithms to a computer at a higher level of abstraction than binary.

Q

Quality attribute:a testable non-functional requirements used to evaluate the performance of a system. Important to an archtect. Import examples are: modifiability, testability, scalability and performance, availability, security and deployability.

Queue:a fundamental data strcture where the item least recently added is removed. Last in last out.

R

Recursion:a function calling itself again and again to complete a task. Router: Only device with a connection to the outside world.

Runtime:The process/length of tiem it takes teh CPU to execute your code

S

Source code:human readable code. What is typed by the programmer into the computer.

Stack:a fundamental data strcture where the item most recently added is removed. Last in first out.

Stream: a long string of bits, e.g. 0111000111.....

Switch:knows MAC addresses for computers on the network, so know which other computer to make a direct physical connection to send data to. Have largely superceded hubs.

T

Testability: the most testable a system the less easily it can hide its faults.

U

Unified Modelling Language (UML): a language for specifying, visualising, constructing, and documenting the artifacts of software systems URL:Unique Resource Locator - unique address to locate and retrive each document.

V

Virtualisation:the creation of a virtual (i.e. not a real) version of something, such as an operating system (OS), a server, a storage device or network resources.
