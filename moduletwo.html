Unit 1 - Introduction to Information Systems

Some information systems support parts of the organisation, others, the whole of the organisation.
Data, Information and Knowledge are key concepts for information systems.
Data (also called 'raw data') is meaningless on its own. Information is when you add 'meaning to data' (i.e. data + meaning = knowledge)
Knowledge is understanding using the information.

Unit 2: Information Systems and their Importance

Shelly et al (2016) suggested three core parts of an information system:
  • The Harware (e.g. computers, keyboard etc...)
  • The software (made up of the OS and Application software) - thse are essentially the instructions that tell the hardware what to do
  • Data
  • People (e.g. IT staff)
  • Processes

It has been suggested that communications should be added to the above. However, technically, Networking belongs in both Hardware and Software.

The above is what separates Information Systems from Computer Science (CS)). CS is one part of Information Systems.

Unit 4: Object-Oriented Development and Python

Object-Oriented programming uses a modular approach, making easier to maintain and ensure flexibility, as blocks can be reused.

A class is like a form/questionnaire. An object is an instance of that class (many instances/objects can be created from a single class).

A class is a blueprint for how something should be achieved, but contains no data itself. In the example below the class of 'dog' specifies its name and age, but does not contain the name of age of any specific dogs. Tips from this week:
